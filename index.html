MokBlok Template Generation Instructions

Prerequisites

You will need a source CSV file containing your translation data set. The CSV should be formatted with each column representing a different word or text element to be translated.

CSV Format Requirements

Proper Format

Your CSV should be structured with:

  • Each column containing a specific word or text element
  • Each row representing a different language or translation variant
  • Headers clearly identifying the content type

Reformatting Your CSV

If your spreadsheet is not in the required format, you will need to transpose your data:

  1. Transposing the Data: If your current CSV has languages as columns and words as rows, you will need to rotate it so that:
    • Words/text elements become column headers
    • Languages/variants become rows
  2. Parsing Additional Data: When transposing, you can extract and reorganize supplementary information such as:
    • Language codes or locale identifiers
    • Context notes or usage guidelines
    • Character limits or formatting requirements
    • Any metadata that should be preserved during the transformation
  3. Tools for Transposition: You can transpose your CSV using:
    • Spreadsheet software (Excel, Google Sheets) using the TRANSPOSE function
    • Programming scripts (Python pandas, R, etc.)
    • Online CSV transformation tools

Understanding CLDR Data Types

Each design element in your Figma template includes a comment specifying which CLDR (Common Locale Data Repository) data types it accepts. For example:

  • A calendar mockup might use: M (Monday), T (Tuesday), W (Wednesday), TH (Thursday), F (Friday), S (Saturday/Sunday)
  • Date formats might use: dd/mm/yyyy or other locale-specific patterns
  • Number formats might specify decimal and thousand separators

Using Figma Buzz Batch Create

Step-by-Step Process

  1. Upload Your CSV: Import your properly formatted CSV file into the Figma Buzz batch create functionality
  2. Select Variable: Choose the specific variable in your Figma template that you want to modify
  3. Map Data Source: Select the corresponding column from your CSV that contains the data for that variable
  4. Generate Mockups: Click the generate button to create mockups based on your selected variables and data mappings

Optimal Use Cases

This application is particularly powerful when:

  • You have multiple entries within a specific column (e.g., translations for many languages)
  • You need to generate numerous design variations simultaneously
  • You are working with batch translations that would otherwise require individual manual updates

The batch create functionality significantly reduces translation time by processing all variants at once rather than requiring individual updates for each language or variation.
